pwInspector(Hydra): trimming passwords


pw-inspector is a tool integrated in Hydra that enables you to prep your word list before launching a guessing attack.
pw-inspector is a tool that trims down word list files to select those words that match a password policy specified by our options.

OPTIONS:
-i → Input file
-o → Output file
-m [n] → Min password length
-M [N] → Max password length
-c [count] → minimum number of password criteria a given word must meet to be included in the list
CRITERIA:
▪ -l → The password must contain at least one lowercase character.
▪ -p → The password must contain at least one printable character that is neither alphabetic nor numeric, which includes !@#$%^&*()
▪ -u → The password must contain at least one uppercase character
▪ -s → The password must include at least one Special character (all others)
▪ -n → The password must contain at least one number

example:
pw-inspector -i /usr/share/seclists/Passwords/Common-Credentials/best1050.txt -o /home/kali/Desktop/custom_wordlist.txt -c 2 -m 6 -n -u -l

the same command can be written as:
cat /usr/share/seclists/Passwords/Common-Credentials/best1050.txt | pw-inspector -m 6 -n -u -l -c 2 > /home/kali/Desktop/custom_wordlist.txt


If we want count the words generated before save it in a file:
cat /usr/share/seclists/Passwords/Common-Credentials/best1050.txt | pw-inspector -m 6 -n -u -l -c 2 | wc -l
