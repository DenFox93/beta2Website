Disable services from being started at boot time



In the AT&T System V (SysV) scheme each service come with a script that can be used to start/stop the service.
While usually each script is supposed to control a single service, some OSs like Solaris could place multiple services in a single script. The important thing anyway is that the they get started in the correct dependency order.


Scripts that start/stop various services are placed usually in /etc/init.d, but this directory could change a little bit from a UNIX OS to another.
