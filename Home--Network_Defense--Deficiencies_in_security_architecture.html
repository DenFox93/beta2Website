Deficiencies in security architecture




Traditional deficiencies in security architecture include:


• Emphasis on perimeter
Emphasis on the security of the perimeter of the Network and assume that the Internal Network(or WAN) is secure.
Only separating systems on VLANs(what are?) does not address the issue. There must also be ACLs (Access Control Lists) firewall rules, etc.
Many organizations ignore basic layer 2 hardening best practices on the flawed assumption that the black hat requires local access to launch them.
If an attacker compromises a local system via a phishing campaign, he could launch layer 2 attacks, including ARP cache poisoning, MAC address spoofing, DHCP (Dynamic Host Configuration Protocol) attacks,...
• Lack of true perimeter (de-perimeterization via cloud/mobile/IoT)
While Classic firewall/perimeter design is often described using a castle analogy: the walls are the firewall, protecting devices on the inside.
Cloud, mobile devices and the Internet of Things (IoT) makes this model (and analogy) far less effective since firewall cannot protect them.
• Prevention Control emphasized
Firewalls, antivirus, patching, host and network-based IPSes (HIPS and NIPS) are primarily preventive controls
Detection controls (IDS) are also important!
• The Internet of Things (IoT)
Usually they lack of patches and Software updates
• Predominantly network-centric
While security on the Network Architecture is important, most recent attacks has moved to the host itseft, precisely on the Application layer.
An Application layer(7) attack could include a phishing email containing a 'poison PDF'.
• Compliance-driven security
While being PCI compliant is undoubtedly a good thing, it's not enough. Organizations must go beyond complying with the letter of the compliance standard.
• We Always Did It This Way
Delay or reject necessary change are not a good thing
• Introducing Technology without Analysis
Introduce tools without careful planning and consideration they could give a false sense of security
