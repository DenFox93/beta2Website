Allowed hosts


/etc/host.allow
nano /etc/hosts.allow

We can set the alowed hosts to connect to the our host
If we want to allow allow all the service available we can set
ALL: [ipAddres(es)Allowed]



/etc/hosts.deny
nano /etc/hosts.deny

by default is better to set all the the hosts disallowed with hosts.deny and then specific which address are allowed with hosts.allow
ALL:ALL


To make effective the changes done we need to restart the services that make use of them (example ssh)

Logs
Is possible to find logs about connections failed ecause of the hardening above (hosts.deny & hosts.allow) on /var/log/auth.log
If an hardening is been done an error like “refuesed connect from” can be found
