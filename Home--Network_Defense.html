Network Defense



Architecture of a Defensible Network(by Richard Bejtlich: link):
• Monitored → Deploy IDSes and IPSes or any sensors that can capture:
◇ session data
◇ full content data (if you can get it)
◇ statistical data
◇ firewall/router/IPS/DNS/proxy/whatever logs
• Inventoried → Know every host and application
• Controlled
◇ ingress filtering
◇ egress filtering
◇ network admission control
◇ network access control
◇ proxy connections
◇ ...
• Claimed
◇ Identify owners of all systems
◇ developing policies, procedures, and plans for the operation of that asset
• Minimized → Reduce the attack surface (can't really implement minimization until you know who owns what)
• Assessed → Conduct vulnerability assessments.
It is possible to place this step before minimization.
Some might argue that it pays to begin with an assessment, but the first question is going to be: "What do we assess?"
